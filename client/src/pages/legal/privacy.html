4. Цели обработки

Оператор обрабатывает персональные данные в следующих целях:

  1. Предоставление пользователю возможности сохранить результат прохождения партии в общем рейтинге.
  2. Формирование именного сертификата прохождения партии для пользователя.
  3. Обеспечение работоспособности и безопасности Сайта (анализ технических данных).
  4. Обработка обращений пользователей по вопросам коммерческого использования Программы.
  5. Исполнение обязательств по заключённым договорам.

5. Правовые основания обработки

Обработка персональных данных осуществляется на следующих правовых основаниях:

  • согласие субъекта персональных данных, выраженное действиями (ввод имени и нажатие кнопки сохранения результата);
  • заключение и исполнение договора, стороной которого является субъект персональных данных (оплата тарифного плана);
  • обеспечение защиты прав и законных интересов оператора (анализ технических данных для информационной безопасности);
  • исполнение функций оператора в соответствии с законодательством Российской Федерации.

6. Способы обработки

Обработка персональных данных осуществляется автоматизированным способом с использованием средств вычислительной техники Оператора. Обработка включает следующие действия:

  • сбор — получение от субъекта при сохранении результата;
  • запись — сохранение в базе данных PostgreSQL на сервере Оператора;
  • хранение — в базе данных в течение срока, указанного в разделе 7;
  • использование — отображение в общем рейтинге участников;
  • удаление — по запросу субъекта или по истечении срока хранения.

Передача персональных данных третьим лицам не осуществляется. Трансграничная передача персональных данных не производится. Все данные хранятся на серверах, расположенных на территории Российской Федерации (Yandex Cloud, зона ru-central1-d).

8. Меры по обеспечению безопасности

Оператор принимает следующие правовые, организационные и технические меры по обеспечению безопасности персональных данных:

  • применение средств криптографической защиты канала передачи данных (TLS 1.3 с сертификатом Let’s Encrypt);
  • хранение базы данных в изолированном сетевом сегменте, доступном только с локального адреса 127.0.0.1;
  • применение системного межсетевого экрана ufw с разрешением только портов 22, 80, 443;
  • применение настроек повышенной безопасности systemd-юнита приложения (NoNewPrivileges, ProtectSystem=strict);
  • хранение паролей доступа к базе данных в защищённом конфигурационном файле с правами доступа 600 и владельцем системного непривилегированного пользователя;
  • ежедневное резервное копирование базы данных с локальным шифрованием в архив gzip и хранением 14 дней;
  • ограничение круга лиц, имеющих доступ к персональным данным, сотрудниками и подрядчиками Оператора, прошедшими согласование с руководителем Оператора;
  • ведение журналов доступа к базе данных средствами PostgreSQL.
